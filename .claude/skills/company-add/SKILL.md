# Company Add — execute pipeline

Runs the finalization pipeline once a batch script (`scripts/add-batch-YYYY-MM-DD.js`) has been generated by `/company-research`.

## Steps (run sequentially)

```bash
# 1. Insert firms into companies.json
node scripts/add-batch-$(date +%Y-%m-%d).js

# 2. Propose ANRE match (via CUI → name+judet → phone)
node scripts/anre-match-companies.js

# 3. Apply match + strip any ANRE-* from certifications[]
node scripts/anre-apply-match.js

# 4. Download logos via Google Favicon + Clearbit (skips firms w/ empty website)
node scripts/download-logos.mjs

# 5. Schema validation
node scripts/company-tools.js validate

# 6. Summary for user
node scripts/company-tools.js stats
```

## Verification before running

- `data/companies.json` is a valid JSON — script will fail otherwise.
- Batch script has no duplicate `id`/`cui` with existing firms — script already logs SKIP.
- `anreMatch.societate` + `judet` must match exactly — `anre-match-companies.js` validates via registry.

## Expected output

- 0 schema errors from `validate` (warnings for missing website/email are OK for no-site firms)
- `stats` shows new count per-county and per-specialization

## On failure

- **Missing specialization**: check `data/specializations.json` for valid IDs (hale-industriale, cladiri-birouri, parcuri-logistice, agricol, retail, hoteluri, spitale, scoli).
- **ANRE match confidence low**: manually set `anreMatch` using exact `societate`/`judet` from `data/anre-atestate.json`.
- **Logo download fails for one firm**: run the site-specific download block manually (the script's built-in Google Favicon fallback usually works for anything with a domain).
